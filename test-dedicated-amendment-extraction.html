🎯 Dedicated Amendment Extraction

Independent useEffect for Amendment Detection

💡 Solution Implemented

Problem: Original amendment extraction logic was blocked by main data loading conditions

Solution: Added dedicated useEffect that runs independently for amendment extraction only

🔄 How It Works:

  • Independent Trigger: Runs separately from main data loading logic
  • Smart Conditions: Only when preloadedData exists, no amendments found, and data is loaded
  • Multi-Method Detection: 3 different amendment detection strategies
  • Performance Safe: Minimal dependencies to prevent re-render loops

📊 Three Detection Methods

Method 1: Status-Based

Filters items with amendment status: pending, submitted, approved, rejected

item.status && ['pending', 'submitted', 'approved', 'rejected'].includes(item.status)

Method 2: Flag-Based

Looks for items with has_amendment flag and amendment_data

item.has_amendment === true && item.amendment_data

Method 3: Quantity-Based

Detects items with amended_qty field

item.amended_qty !== null && item.amended_qty !== undefined

🚨 Troubleshooting

If no AmendmentExtraction logs appear:

  1. Check that preloadedData exists
  2. Verify pendingAmendments.length is 0
  3. Ensure isDataLoaded is true
  4. Look for the useEffect trigger conditions

If logs appear but no amendments found:

  1. Check the data structure of storesWithSubmissions
  2. Verify that the 11 amendments have the expected fields
  3. Look at the sample items logged to see their structure
  4. Consider that amendments might be in a different format
